# Voice Settings

OrtaVox exposes a range of voice settings that control how the agent speaks and listens. Some settings are provider-specific (e.g. ElevenLabs voice parameters), while others are cross-provider and apply to the conversation management layer regardless of which TTS or STT you use.

---

## Voice Quality Parameters

### `voice.speed`

Controls how fast the agent speaks.

| Property | Value |
|---|---|
| Type | `float` |
| Range | `0.5` – `2.0` |
| Default | `1.0` |
| Providers | All |

`1.0` is the natural speaking rate of the chosen voice. Values below `1.0` slow speech down; values above `1.0` speed it up. Most callers find values in the range `0.9` – `1.15` to be comfortable for support conversations.

```json
{
  "voice": {
    "provider": "elevenlabs",
    "voiceId": "dqzVFKHwZvR9YMQoFM67",
    "speed": 1.1
  }
}
```

<Tip>
For outbound sales or appointment reminder calls, a slightly faster rate (1.05–1.1) can feel more natural and energetic. For elderly callers or complex instructions, slow down to 0.85–0.95.
</Tip>

---

### `voice.stability` (ElevenLabs only)

Controls how consistent the voice sounds across sentences and turns.

| Property | Value |
|---|---|
| Type | `float` |
| Range | `0.0` – `1.0` |
| Default | `0.75` |
| Providers | `elevenlabs` |

At lower values (`0.0`–`0.4`) the voice is more expressive and dynamic but may sound inconsistent between turns. At higher values (`0.7`–`1.0`) the voice is stable and predictable, which works well for professional or neutral-tone agents.

```json
{
  "voice": {
    "provider": "elevenlabs",
    "voiceId": "dqzVFKHwZvR9YMQoFM67",
    "stability": 0.80
  }
}
```

---

### `voice.similarityBoost` (ElevenLabs only)

Controls how closely the synthesized voice adheres to the original voice clone's characteristics.

| Property | Value |
|---|---|
| Type | `float` |
| Range | `0.0` – `1.0` |
| Default | `0.75` |
| Providers | `elevenlabs` |

Higher values increase similarity to the source voice but can amplify audio artifacts in the original voice sample. For cloned voices with high-quality samples, values of `0.80`–`0.90` work well. For pre-built voices, `0.70`–`0.80` is a good default.

```json
{
  "voice": {
    "provider": "elevenlabs",
    "voiceId": "dqzVFKHwZvR9YMQoFM67",
    "similarityBoost": 0.80
  }
}
```

---

### `voice.style` (ElevenLabs only)

Controls the expressiveness style intensity of the voice.

| Property | Value |
|---|---|
| Type | `float` |
| Range | `0.0` – `1.0` |
| Default | `0.0` |
| Providers | `elevenlabs` |

At `0.0` the voice uses its trained neutral delivery. Higher values exaggerate the emotional character of the voice model. Use sparingly: values above `0.5` can sound theatrical or unnatural for most support contexts.

```json
{
  "voice": {
    "provider": "elevenlabs",
    "voiceId": "dqzVFKHwZvR9YMQoFM67",
    "style": 0.2
  }
}
```

---

### `voice.useSpeakerBoost` (ElevenLabs only)

Applies an audio processing pass that boosts vocal clarity and perceived presence.

| Property | Value |
|---|---|
| Type | `boolean` |
| Default | `false` |
| Providers | `elevenlabs` |

When enabled, the voice sounds "closer" and more present — useful for phone calls where audio fidelity is reduced by codec compression. Adds a small amount of processing latency.

```json
{
  "voice": {
    "provider": "elevenlabs",
    "voiceId": "dqzVFKHwZvR9YMQoFM67",
    "useSpeakerBoost": true
  }
}
```

## Conversation Behavior Settings

These settings are defined at the top level of the agent configuration (not inside `voice`). They control how the agent manages the flow of conversation.

### `backchannels`

Enables the agent to speak short acknowledgment sounds while the user is still talking — "mm-hmm", "I see", "right" — to signal it is listening. This makes conversations feel significantly more natural and human.

| Property | Value |
|---|---|
| Type | `boolean` |
| Default | `false` |
| Providers | All |

```json
{
  "backchannels": true
}
```

<Note>
Backchannels are generated by a separate small model and do not interrupt the main conversation flow. They are not transcribed and do not appear in call transcripts.
</Note>

---

### `interruptionSensitivity`

Controls how aggressively the agent stops speaking when the user begins to talk.

| Property | Value |
|---|---|
| Type | `float` |
| Range | `0.0` – `1.0` |
| Default | `0.5` |
| Providers | All |

| Value | Behavior |
|---|---|
| `0.0` | Agent never interrupts itself mid-sentence, regardless of user speech. |
| `0.3` | Agent continues speaking unless the user speaks clearly and sustained. |
| `0.5` | Balanced default. Agent yields when it detects clear user intent to speak. |
| `0.8` | Agent yields at the first sign of user speech, even partial syllables. |
| `1.0` | Maximum sensitivity: agent stops at any audio above the noise floor. |

```json
{
  "interruptionSensitivity": 0.6
}
```

<Warning>
Setting `interruptionSensitivity` too high (above 0.8) can cause the agent to constantly interrupt itself due to background noise, breathing sounds, or filler words. Test in the environments your callers will actually use.
</Warning>

---

### `reminderAfterSilenceMs`

After the user has been silent for this many milliseconds without ending the call, the agent speaks a gentle reminder prompt ("Are you still there?") to re-engage them.

| Property | Value |
|---|---|
| Type | `integer` |
| Unit | Milliseconds |
| Default | `null` (disabled) |
| Providers | All |

```json
{
  "reminderAfterSilenceMs": 8000,
  "reminderMessage": "Sorry, I didn't catch that. Could you repeat what you said?"
}
```

If the user does not respond within `endCallAfterSilenceMs` of their last speech, the call ends normally. `reminderAfterSilenceMs` should be set lower than `endCallAfterSilenceMs`.

---

### `beginMessageDelayMs`

Delays the agent's first message by this many milliseconds after the call connects. Useful for inbound phone calls where there may be a brief audio handshake period, or for web calls where you want a moment before the agent begins speaking.

| Property | Value |
|---|---|
| Type | `integer` |
| Unit | Milliseconds |
| Default | `0` |
| Range | `0` – `5000` |
| Providers | All |

```json
{
  "firstMessage": "Hi, thanks for calling. How can I help you today?",
  "beginMessageDelayMs": 1000
}
```

---

### `ambientSound`

Plays a low-volume background audio track behind the agent's voice during the call. This increases perceived naturalness and masks silence gaps between turns.

| Property | Value |
|---|---|
| Type | `string` |
| Default | `null` (disabled) |
| Providers | All |

Available options:

| Value | Description |
|---|---|
| `coffee-shop` | Quiet café ambient noise: background chatter, soft clink of cups |
| `office` | Open-plan office: keyboard clicks, quiet hum |
| `call-center` | Busy call center floor: low background activity |
| `nature` | Gentle outdoor nature sounds |
| `none` | Explicitly disabled (same as `null`) |

```json
{
  "ambientSound": "office"
}
```

<Tip>
The `call-center` ambient sound can help frame the call context for callers and reduce the uncanny valley effect of a perfectly silent background on phone calls, where callers expect some background noise.
</Tip>

## Pronunciation Dictionary

The pronunciation dictionary lets you override how specific words, acronyms, or proper nouns are spoken. This is useful for brand names, technical terms, abbreviations, and unusual proper nouns that TTS engines tend to mispronounce.

Overrides are defined as a list of objects with a `word` and a `phoneme` (using IPA notation) or a `substitution` (a simpler spelled-out pronunciation hint).

### Using Substitution (Simpler)

Substitution replaces the word with a phonetically spelled alternative that the TTS engine handles correctly. No IPA knowledge required.

```json
{
  "pronunciationDictionary": [
    { "word": "OrtaVox", "substitution": "Or-tah-vox" },
    { "word": "TTFB",    "substitution": "T-T-F-B" },
    { "word": "SQL",     "substitution": "sequel" },
    { "word": "PyPI",    "substitution": "pie-pee-eye" }
  ]
}
```

### Using IPA Phoneme (Precise)

IPA notation gives you precise control over pronunciation when substitution is not accurate enough.

```json
{
  "pronunciationDictionary": [
    { "word": "Ortavox", "phoneme": "ˌɔːrtəˈvɒks" },
    { "word": "cache",   "phoneme": "kæʃ" }
  ]
}
```

<Note>
Pronunciation dictionary overrides apply globally to all TTS synthesis for that agent. They are case-insensitive and match whole words only (not substrings).
</Note>

## Normalize for Speech

When enabled, OrtaVox automatically converts numeric and symbolic content into spoken-word equivalents before passing text to the TTS provider. This prevents TTS engines from reading out characters literally (e.g., "$150.00" read as "dollar one five zero point zero zero").

| Property | Value |
|---|---|
| Type | `boolean` |
| Default | `true` |
| Providers | All |

When `normalizeForSpeech` is `true`, the following conversions happen automatically:

| Input text | Spoken as |
|---|---|
| `$150.00` | "one hundred and fifty dollars" |
| `3pm` | "three PM" |
| `10/24/2026` | "October twenty-fourth, twenty twenty-six" |
| `https://ortavox.ai` | "ortavox dot ai" |
| `+1 (415) 555-0100` | "plus one four one five five five five zero one zero zero" |
| `API` | "A P I" |

```json
{
  "normalizeForSpeech": true
}
```

Set to `false` only if your LLM system prompt already produces fully spoken-word text, or if you are using a TTS provider that handles normalization natively.

## Full Voice Settings Example

```json
{
  "name": "Premium Support Agent",
  "transcriber": {
    "provider": "deepgram",
    "model": "nova-3",
    "language": "en"
  },
  "model": {
    "provider": "openai",
    "model": "gpt-4o",
    "systemPrompt": "You are a helpful, warm support agent for Acme Corp."
  },
  "voice": {
    "provider": "elevenlabs",
    "voiceId": "dqzVFKHwZvR9YMQoFM67",
    "speed": 1.0,
    "stability": 0.75,
    "similarityBoost": 0.85,
    "style": 0.15,
    "useSpeakerBoost": true
  },
  "backchannels": true,
  "interruptionSensitivity": 0.55,
  "beginMessageDelayMs": 800,
  "reminderAfterSilenceMs": 8000,
  "reminderMessage": "Are you still there? Take your time.",
  "endCallAfterSilenceMs": 25000,
  "ambientSound": "office",
  "normalizeForSpeech": true,
  "pronunciationDictionary": [
    { "word": "OrtaVox", "substitution": "Or-tah-vox" },
    { "word": "API",     "substitution": "A-P-I" }
  ]
}
```

## Settings Quick Reference

| Setting | Type | Default | Scope |
|---|---|---|---|
| `voice.speed` | float `0.5`–`2.0` | `1.0` | All providers |
| `voice.stability` | float `0`–`1` | `0.75` | ElevenLabs only |
| `voice.similarityBoost` | float `0`–`1` | `0.75` | ElevenLabs only |
| `voice.style` | float `0`–`1` | `0.0` | ElevenLabs only |
| `voice.useSpeakerBoost` | boolean | `false` | ElevenLabs only |
| `backchannels` | boolean | `false` | All providers |
| `interruptionSensitivity` | float `0`–`1` | `0.5` | All providers |
| `reminderAfterSilenceMs` | integer (ms) | disabled | All providers |
| `reminderMessage` | string | — | All providers |
| `beginMessageDelayMs` | integer `0`–`5000` | `0` | All providers |
| `ambientSound` | string enum | disabled | All providers |
| `pronunciationDictionary` | array | `[]` | All providers |
| `normalizeForSpeech` | boolean | `true` | All providers |
